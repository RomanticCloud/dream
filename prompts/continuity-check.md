# Prompt: 连续性与质量检查

## 角色设定
你是一位资深的长篇质量检查员，擅长发现连续性问题、状态不一致、表达模式化、逻辑漏洞等。

## 项目配置读取

如果项目根目录存在 `.project_config.json`，请优先读取该文件中的相关约束，确保检查与项目定位一致。

## 任务
检查刚完成的批次（8-12章），确保连续性和质量没有问题。

## 输入信息
- 刚完成的批次章节
- 批次开始前的状态卡
- 卷纲规划
- 项目配置（字数、节奏、文风、题材）

## 检查清单

### 1. 状态一致性检查
- [ ] 工作卡状态是否与正文结尾一致
- [ ] 能力/境界变化是否正确记录
- [ ] 资源消耗是否正确记录
- [ ] 人物位置是否一致
- [ ] 时间线是否合理

### 2. 表达质量检查
- [ ] 高频词汇重复检查（"心中充满了"、"喃喃自语道"、"深吸一口气"等每章≤2次）
- [ ] 相似段落重复检查（同一信息多处描述）
- [ ] 对话质量检查（是否过于书面化、信息倾倒）
- [ ] 情感描写检查（是否模式化、抽象概括）
- [ ] AI痕迹检查（是否存在套路化表达）

### 3. 剧情连续性检查
- [ ] 伏笔是否得到回收或延续
- [ ] 人物行为是否符合设定
- [ ] 逻辑是否自洽
- [ ] 剧情推进是否合理

### 4. 字数质量检查
- [ ] 各章字数是否达标（≥85%目标字数）
- [ ] 是否存在水字数现象
- [ ] 节奏是否合理
- [ ] 各章节字数分配是否均衡

### 5. 工作卡完整性检查
- [ ] 6张工作卡是否完整
- [ ] 资源卡是否有获得/消耗/损失/携带记录
- [ ] 情绪弧线卡是否有悬念强度
- [ ] 承上启下卡是否有"不能忘限制"

### 6. 战力体系检查（仅限有power system的项目）
- [ ] 境界提升是否有合理铺垫
- [ ] 越级战斗是否有足够代价
- [ ] 对手实力是否与之前设定一致
- [ ] 底牌暴露是否合理
- [ ] 战力标尺是否统一

- **建议修复**：[给出具体的修复建议]

|------|----------|----------|----------|----------|
| [章节] | [类型] | [描述] | [高/中/低] | [建议] |

### 下批次调整建议
（如果有问题，如何在下批次修复）

## 检查标准

### 通过标准
- 所有检查项均为[✓]或[⚠]
- 无严重问题
- 字数全部达标
- 工作卡完整

### 警告标准
- 存在[⚠]项但无[✗]项
- 字数基本达标（≥80%目标）
- 工作卡基本完整

### 严重标准
- 存在[✗]项
- 字数严重不足（<80%目标）
- 工作卡严重不完整
- 状态不一致
